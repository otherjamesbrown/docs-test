##### Using Regular Expression

Regular expression, also known as _regex_ or _regexp_, is a standard pattern matching tool used in many scripting languages. It allows you to create filters that can match patterns of text, rather than just single words or phrases.

Content filtering in SpamTitan uses Perl Compatible Regular Expression (PCRE) regular expressions. For more information on regular expression and PCRE see [www.regular-expressions.info](https://www.regular-expressions.info/) ![icons8-external-link-32.png](/docs-test/imported-assets/spamtitan-legacy/8952-using-regular-expression/9407e2037883.png) and [www.pcre.org](https://www.pcre.org/) ![icons8-external-link-32.png](/docs-test/imported-assets/spamtitan-legacy/8952-using-regular-expression/9407e2037883.png).

Always test your regular expressions for matching and false positives using a testing tool like [regexpal.com](http://regexpal.com/) ![icons8-external-link-32.png](/docs-test/imported-assets/spamtitan-legacy/8952-using-regular-expression/9407e2037883.png).

Wildcards

Wildcards are an important part of regular expressions. These are different from the wildcards used in SQL or file system wildcards. Below are some common ones you may use:

<table><thead><tr><th><p>Wildcard</p></th><th><p>Description</p></th></tr></thead><tbody><tr><td><p>.</p></td><td><p>matches any character, except a newline (includes spaces, tabs, letters, and numbers).</p></td></tr><tr><td><p>\d</p></td><td><p>matches any digit (0-9).</p></td></tr><tr><td><p>\s</p></td><td><p>matches whitespace (space or tabs).</p></td></tr><tr><td><p>\w</p></td><td><p>matches any word character (a-z and A-Z)</p></td></tr><tr><td><p>[ ]</p></td><td><p>matches any character within the brackets. For example, [b,t,z] will match b, t, or z. [a-z0-9] will match a to z <span><em>or</em></span> 0 to 9.</p></td></tr><tr><td><p>\</p></td><td><p>use as an "escape" to strip special properties of any character. For example, if you want a period, use "\." instead of ".".</p></td></tr><tr><td><p>(|)</p></td><td><p>matches any of the patterns inside the parentheses. For example, (\.tw|\.jp) matches ".tw" or ".jp".</p></td></tr><tr><td><p>*</p></td><td><p>matches zero or more of the preceding element or pattern.</p></td></tr><tr><td><p>+</p></td><td><p>matches one or more of the preceding element or pattern.</p></td></tr><tr><td><p>?</p></td><td><p>matches zero or one of the preceding element or pattern.</p></td></tr><tr><td><p>{n}</p></td><td><p>matches exactly "n" number of the previous character or pattern.</p></td></tr><tr><td><p>{n,m}</p></td><td><p>matches between "n" and "m" number of the previous character or pattern.</p></td></tr><tr><td><p>{n,}</p></td><td><p>matches at least "n" of the previous character or pattern.</p></td></tr><tr><td><p>^</p></td><td><p>matches the beginning of a string.</p></td></tr><tr><td><p>$</p></td><td><p>matches the end of a string.</p></td></tr><tr><td><p>\b</p></td><td><p>a word break, which matches the beginning {\&amp;lt;} or end {\&amp;gt;} of a word. It is NOT a catchall for tabs, spaces, etc. between words.</p></td></tr></tbody></table>
