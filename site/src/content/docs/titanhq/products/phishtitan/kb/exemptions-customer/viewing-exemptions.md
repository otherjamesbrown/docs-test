Go to **Email Security** > **Configuration** > **Exemptions** to view and manage Exemptions.

In the All tab, which appears by default, you'll see the complete list of exemptions that exist for specific email addresses, domains, and IPs. To see Exemptions based on each type, select the relevant tab.

![ES-Exemptions-Customers-All-List1.jpg](/docs-test/imported-assets/phishtitan-kb/4000228160/50a67eb8246e.jpg)

-   Use the search ![Search-Box.jpg](/docs-test/imported-assets/phishtitan-kb/4000228160/ec5774fc0179.jpg) box to search the list.

-   Select the column ![Column-icon.jpg](/docs-test/imported-assets/phishtitan-kb/4000228160/87a2adaac243.jpg) icon to manage how table columns are displayed. By default, columns are arranged in a specific order. You can change the display by doing the following:

    -   Select or deselect the checkbox beside a column name to show or hide a column. Note that the _Created Date_ and _Modified By_ columns are not visible by default.

    -   Use the reordering![Reordering\_icon.jpg](/docs-test/imported-assets/phishtitan-kb/4000228160/317808a87dea.jpg) icon to move columns up and down the list. This changes the left-to-right display. An item at the top of the list, for example, means that it will be the first column on the left. Select **Reset Order** to restore the columns to their default positions.

-   Resize columns by hovering your cursor over the column boundary you want to adjust until it becomes a resize ![ES-Resize-Cursor.jpg](/docs-test/imported-assets/phishtitan-kb/4000228160/3b077003c7a7.jpg) cursor. Drag the boundary until the column is the width you want.

The following fields are available:

-   _Address_: This is the email address, domain, or IP address for which the exemption was created.

-   _Type_: Indicates if it is an email address, domain, or IP address.

-   _Bypass Checks_: The security checks that were selected to be bypassed, to prevent an email from being flagged or auto-remediated.

-   _Comments_: Optional description that was entered when the Exemption was added.

-   _Source_: Indicates if the entry is inherited from the MSP level. If the entry is only allowed on the customer level, it will display as _Local_.

-   _Last Modified_: Date that the Exemption entry was last edited. If it has been edited, then the date automatically updates.

-   _Created Date_: Date that the Exemption was created. Note that this column is not visible by default; select the column ![Column-icon.jpg](/docs-test/imported-assets/phishtitan-kb/4000228160/87a2adaac243.jpg) icon to enable it.

-   _Modified By_: Email address of the person who last modified the Exemption. Note that this column is not visible by default; select the column ![Column-icon.jpg](/docs-test/imported-assets/phishtitan-kb/4000228160/87a2adaac243.jpg) icon to enable it.

-   _Actions_: From the Actions column, you can edit or delete the email address or domain.

    :::note
    Exemptions added on the MSP level cannot be edited or deleted on the customer level.
    :::

    -   Select the edit ![edit-icon.jpg](/docs-test/imported-assets/phishtitan-kb/4000228160/90a487f7499b.jpg) icon beside the Exemption you want to edit. You can edit the Address, Bypass Checks, and Comments as described above.

    -   Select the delete ![PT-Delete-Icon.jpg](/docs-test/imported-assets/phishtitan-kb/4000228160/1c64a3c2643d.jpg) icon beside the Exemption you want to delete. Select **Delete** to confirm.
